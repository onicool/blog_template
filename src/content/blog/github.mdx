## git、githubとは
- git：バージョン管理システム

    同じファイルを続けて編集するときに、前のバージョンを残しておける。完成版最終版改訂版などといったファイルを作らなくて済む。

- github：gitを使っているプロジェクトをホスティングするサービス

    オンラインで保存してgitを複数端末或いは複数人で使える。


## インストール
- git：https://git-scm.com/

    インストーラの案内に従ってインストール<br/>
    全部初期設定でOK。自分はブランチ名をmasterからmainに変更した（どっちでもいい。名前の違い）。<br/>
    ＞ブランチとは：開発ライン。開発を分担する場合などに別のブランチを作って割り当てる。私は今のところ一人なのでmainだけ。

- github：https://github.com/

    アカウントとリポジトリ作成<br/>
    ＞リポジトリとは：プロジェクトのこと。ファイルやフォルダをまとめて保存する場所。

    

## 初期設定～PUSHまで
開発中は３から５を繰り返す。<br/>
ローカルで保存するだけの時は３と４だけ。

1. リポジトリ作成（ローカルでgit管理を開始する）
    ```
    git init
    ```
2. リモートリポジトリとの紐づけ（オンラインのgithubとの紐づけ）
    ```
    git remote add origin https://XXXXX
    ```
3. ファイル追加（リポジトリに入れるファイルを選ぶ。まだローカル）
    ```（ファイル全部の場合）
    git add .
    ```
4. コミット（リポジトリに記録。まだローカル）
    ```
    git commit -m "first commit"
    ```
5. プッシュ（リモートリポジトリにアップロード）
    ```
    git push -u origin main
    ```
    ２回目からはorigin mainを省略可
    ```
    git push
    ```
